# Политика слияния при нормализации

Версия: **2.1.0** (canon 2.2.0)

Нормализация — **слияние** с существующим репозиторием, не замена. Агент следует канону и чеклисту роли, сохраняя ценное в проекте.

См. `normalize-governance.md`, `normalize.bundle.yaml`.

---

## Принципы

1. Добавлять и обновлять то, что требует канон; лишнее в целевом репо не удалять.
2. Учитывать контекст проекта: продуктовый код, docs и зависимости остаются осмысленными для этого репо.
3. `protocol-ref/` появляется при protocol reconcile (group-sync), не при первичном normalize.

---

## Маркер `project-local:`

```markdown
<!-- project-local: -->
```
