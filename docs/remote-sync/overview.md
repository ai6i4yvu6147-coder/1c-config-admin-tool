## Remote Sync — обзор и продуктовое видение

### Проблема

- Много клиентов, доступ через **RDP**.
- Выгрузки XML делаются (или будут делаться) **на удалённой машине**.
- **config-mcp** установлен **только локально** — индекс строится у вас.
- Боль: **ручное копирование** больших каталогов XML через RDP (обрывы, медленно, нестабильно).

### Цель

Автоматизировать **доставку** уже существующих (или будущих) XML с RDP в локальный `{ExportRoot}` ConfigAdmin, далее — существующий pipeline (`apply-registry` → config-mcp).

**Не в scope первого E2E:** запуск Designer/выгрузка 1С на RDP из Hub (это Phase R2).

---

## Один универсальный exe, два режима

**Решение:** один артеfact сборки (`ConfigAdmin.exe` в `dist/wpf-single/`), при запуске — **выбор режима**:

| Режим | Где запускается | Назначение |
|-------|-----------------|------------|
| **Админка (Hub)** | Ваш локальный ПК | Профили, базы, MCP, постановка задач sync, приём файлов |
| **Передатчик (Sync Agent)** | RDP клиента | Pairing с Hub, poll задач, чтение `remote_source_path`, upload с докачкой |

### Поведение при старте

```
ConfigAdmin.exe
  ├─ [ Админка ]     → текущий WPF (vault → главный экран)
  └─ [ Передатчик ]  → минимальный UI: URL Hub, ID узла, pairing-пароль, статус, лог
```

**Сохранение выбора (опционально):** «Запомнить режим на этом компьютере» → `appsettings.local.json` (`mode: hub | agent`).

**Headless / служба (Phase R2):** тот же бинарник:

```text
configadmin.exe sync-agent --hub https://... --node-id ... 
```

Для MVP достаточно WPF-режима «Передатчик»; console — для планировщика и отладки.

### Почему один exe, а не два продукта

- Общий код: vault (при необходимости), paths, manifest, logging, transport client/server.
- Один portable для RDP: скопировал папку `dist/wpf-single/` — выбрал «Передатчик».
- Согласуется с Admin Hub protocol: `moduleType: config-admin`, режим `hub | sync-agent`.

## Протокол передачи — выбор

### Требования

1. **Работает на типичном RDP Windows** без прав администратора на установку служб/проброса портов **на стороне RDP**.
2. **Исходящее соединение** с RDP (outbound) — Hub часто за NAT; inbound на ваш ПК с интернета ненадёжен.
3. **Докачка** при обрыве (resume).
4. **Проверка целостности** (SHA-256 всего payload).
5. **HTTPS** — проходит через корпоративные прокси/FW чаще, чем произвольные порты.

### Выбранный протокол: **HTTPS Chunk Upload v1**

Normative spec: [`transport.md`](transport.md).

| Альтернатива | Почему не primary |
|--------------|-------------------|
| SFTP/FTP на Hub | Hub должен **слушать** входящие; на домашнем/офисном ПК — проброс портов, dynamic IP |
| RDP drive mapping | Ручной, без resume на уровне приложения |
| SMB share | Часто закрыт между сетями клиента и вами |
| S3 промежуточное | Хорошо для async (Phase R3), избыточно для MVP |

**Outbound-only:** передатчик на RDP **сам подключается** к Hub (poll + upload). Локально вы нажимаете «Синхронизировать» → задача в очереди → agent забирает при следующем poll.

### Доступность Hub с RDP

Если Hub не виден из интернета напрямую:

| Вариант | Когда |
|---------|--------|
| **Tailscale Funnel** (используется сейчас) | Hub за NAT, RDP без VPN; outbound HTTPS |
| Tailscale mesh (`100.x.x.x`) | Tailscale на обеих сторонах |
| Relay VPS (Phase R3) | Много клиентов, несколько рабочих ПC |

Transport (HTTPS chunks) **не меняется** — меняется только базовый URL Hub.

**Инструкция:** [`network-setup.md`](network-setup.md). **Статус:** [`status.md`](status.md).

## Первый E2E (что вы хотите увидеть)

1. Локальная **Админка**: клиенты, базы, привязка **удалённый узел (RDP)**.
2. У базы: **Local / Remote**, для Remote — узел + **абсолютный путь на RDP**.
3. На RDP: **Передатчик**, pairing-пароль совпадает с настройками узла в Hub.
4. В Админке: **«Синхронизировать с RDP»** → файлы в локальном `{ExportRoot}/.../Основная конфигурация`.
5. Опционально: **«Sync с MCP»** (уже реализовано).

Детали: [`mvp-spec.md`](mvp-spec.md).

## Три типа паролей (не смешивать в UI)

| Пароль | Режим | Назначение |
|--------|-------|------------|
| Мастер-vault | Админка (+ позже Передатчик при export) | Шифрование паролей **баз 1С** |
| Pairing-пароль узла | Оба | Аутентификация **Передатчик ↔ Hub** |
| Пароль базы 1С | Админка | Designer / export (не нужен для MVP copy-only) |

На MVP (только копирование файлов) на RDP **vault не обязателен** — только pairing.
