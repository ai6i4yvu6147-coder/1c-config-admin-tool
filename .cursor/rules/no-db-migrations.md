# NO_DB_MIGRATIONS

В проекте ConfigAdmin **запрещены миграции существующих SQLite-баз** (`configadmin.db`).

## Не делать

- `ALTER TABLE` для обновления уже развёрнутых БД пользователей
- скрипты конвертации данных «старой схемы → новой»
- автоматический перенос профилей/vault между версиями схемы

## Допустимо

- `CREATE TABLE IF NOT EXISTS` при **первом** создании пустого файла БД (bootstrap)
- добавление новых таблиц только в bootstrap для **новых** установок

## При несовместимых изменениях схемы

Пользователь удаляет `%AppData%\ConfigAdmin\` или указывает новый `--db` и настраивает профили заново. Выгруженные XML в `{ExportRoot}` не удаляются.
